# Java 練習問題：給与計算システム

## 概要

複数種類の従業員をまとめて扱う給与計算システムを作成してください。

この問題では、従業員に共通する情報と、従業員の種類ごとに異なる処理を分けて設計します。

---

## パッケージ

すべてのクラス・インターフェースは次のパッケージに配置すること。

```java
package training.original.o03;
```

---

## 作成するファイル

以下を作成すること。

```text
Employee
RegularEmployee
PartTimeEmployee
Manager
BonusEligible
PayrollService
Main
```

---

## 制約事項

以下は使用しないこと。

```text
List
Stream API
record
Lombok
```

配列、`for` 文、`if` 文、通常のクラス定義を使って実装してください。  
必要に応じて拡張for文は使ってよいです。

---

# 1. Employee

## 役割

`Employee` は、従業員に共通する情報と操作を表す基底クラスです。

ただし、`Employee` だけでは具体的な月給計算方法や役割名が決まりません。  
そのため、`Employee` 自体をそのままインスタンス化する対象にはしない設計にしてください。

## 保持する情報

```text
従業員ID
名前
```

外部からフィールドを直接変更できないようにしてください。

## 生成時のルール

```text
id <= 0 の場合:
  IllegalArgumentException

name が null または空白のみの場合:
  "unknown"
```

## 提供する操作

```text
従業員IDを取得する
名前を取得する
名前を変更する
月給を計算する
役割名を取得する
従業員情報を表示する
自分と同じ内容を持つ別インスタンスを返す
```

## 名前の変更

名前を変更するときも、生成時と同じ名前補正ルールを適用してください。

## 表示

従業員情報を標準出力に表示してください。

表示形式は自由ですが、少なくとも以下が確認できるようにしてください。

```text
id
name
role
monthlyPay
```

---

# 2. BonusEligible

## 役割

`BonusEligible` は、ボーナス計算の対象になる従業員が持つ性質を表します。

従業員の種類とは別に、「ボーナスを計算できる」という性質を表せるようにしてください。

## 提供する操作

```text
ボーナス額を計算する
```

---

# 3. RegularEmployee

## 役割

`RegularEmployee` は正社員を表します。

正社員は従業員であり、ボーナス対象者でもあります。

## 保持する情報

```text
基本給
残業時間
```

## 生成時のルール

```text
baseSalary < 0 の場合:
  0

overtimeHours < 0 の場合:
  0
```

## 月給

以下の式で計算します。

```text
baseSalary + overtimeHours * 2000
```

## 役割名

```text
正社員
```

## ボーナス

以下の式で計算します。

```text
baseSalary / 10
```

## コピー

自分と同じ内容を持つ別インスタンスを返せるようにしてください。

---

# 4. PartTimeEmployee

## 役割

`PartTimeEmployee` はアルバイトを表します。

アルバイトは従業員ですが、この問題ではボーナス対象者ではありません。

## 保持する情報

```text
時給
勤務時間
```

## 生成時のルール

```text
hourlyWage < 0 の場合:
  0

workingHours < 0 の場合:
  0
```

## 月給

以下の式で計算します。

```text
hourlyWage * workingHours
```

## 役割名

```text
アルバイト
```

## コピー

自分と同じ内容を持つ別インスタンスを返せるようにしてください。

---

# 5. Manager

## 役割

`Manager` は管理職を表します。

管理職は従業員であり、ボーナス対象者でもあります。

## 保持する情報

```text
基本給
管理職手当
```

## 生成時のルール

```text
baseSalary < 0 の場合:
  0

managementAllowance < 0 の場合:
  0
```

## 月給

以下の式で計算します。

```text
baseSalary + managementAllowance
```

## 役割名

```text
管理職
```

## ボーナス

以下の式で計算します。

```text
baseSalary / 5
```

## 固有操作

管理職には、会議を開催する操作を用意してください。

実行すると、会議を開催する旨のメッセージを標準出力に表示します。

## コピー

自分と同じ内容を持つ別インスタンスを返せるようにしてください。

# 6. PayrollService

## 役割

`PayrollService` は、複数の従業員をまとめて管理し、給与やボーナスを集計するクラスです。

## 保持する情報

```text
Employee の配列
```

外部からフィールドを直接変更できないようにしてください。

---

## 生成時のルール

`Employee` 配列を受け取り、以下のルールで内部に保存してください。

```text
引数が null の場合:
  長さ0の配列として扱う

配列内に null 要素がある場合:
  除外する

有効な従業員だけを、隙間なく詰めた配列として保持する

受け取った従業員をそのまま保持せず、それぞれ別インスタンスとして保持する
```

`PayrollService` は、正社員・アルバイト・管理職といった具体的な種類をなるべく意識せずに扱えるようにしてください。

---

## 提供する操作

```text
全従業員の月給合計を取得する
ボーナス対象者のボーナス合計を取得する
全従業員の情報を表示する
ボーナス対象者の名前とボーナス額を表示する
```

---

## 月給合計

管理している全従業員の月給合計を返してください。

従業員の種類ごとの月給計算に従って合計してください。

---

## ボーナス合計

ボーナス対象者だけを対象に、ボーナス合計を返してください。

ボーナス対象ではない従業員は合計対象に含めません。

---

## 全件表示

全従業員の情報を順番に表示してください。

---

## ボーナス対象者の表示

ボーナス対象者だけについて、名前とボーナス額を表示してください。

---

# 7. Mainでの動作確認

以下のような従業員配列を作成し、`PayrollService` に渡して動作確認してください。

```java
Employee[] employees = {
    new RegularEmployee(1, "Taro", 300000, 10),
    new PartTimeEmployee(2, "Hanako", 1200, 80),
    new Manager(3, "Sato", 500000, 100000),
    null
};
```

最低限、以下を確認してください。

```text
全従業員の情報表示
月給合計
ボーナス合計
ボーナス対象者の表示
null要素が除外されていること
```

期待される月給の例：

```text
Taro:
  300000 + 10 * 2000 = 320000

Hanako:
  1200 * 80 = 96000

Sato:
  500000 + 100000 = 600000

月給合計:
  1016000
```

期待されるボーナスの例：

```text
Taro:
  300000 / 10 = 30000

Sato:
  500000 / 5 = 100000

ボーナス合計:
  130000
```

# 8. ポリモーフィズムの確認

`Employee` 型の変数に `Manager` インスタンスを代入して、以下を確認してください。

```java
Employee e = new Manager(4, "Suzuki", 600000, 120000);
```

確認すること。

```text
Employee 型の変数から、共通操作を呼び出せること
実行時には Manager の月給計算・役割名が使われること
Employee 型のままでは Manager 固有操作を直接呼び出せないこと
```

必要であれば、型を確認してから `Manager` として扱う処理も試してください。

---

# 9. 外部データからの独立

`PayrollService` 生成後に、元の配列や元の従業員オブジェクトを変更しても、`PayrollService` 内部のデータが影響を受けないことを確認してください。

確認例：

```java
RegularEmployee regular = new RegularEmployee(1, "Taro", 300000, 10);
Manager manager = new Manager(2, "Sato", 500000, 100000);

Employee[] employees = {regular, manager};

PayrollService service = new PayrollService(employees);

regular.setName("Changed");
manager.setName("ChangedManager");
employees[0] = new PartTimeEmployee(99, "External", 1000, 1);

service.printAll();
```

確認すること。

```text
PayrollService 内部の従業員名が生成時の状態を保っている
配列差し替えの影響を受けていない
```

# 11. 実装後の確認事項

実装後、次の観点を説明できるようにしてください。

```text
1. 従業員の共通部分と種類ごとの差分をどのように分けたか。
2. 正社員・アルバイト・管理職を同じ配列で扱える理由。
3. Employee 型の変数に Manager を入れたとき、月給計算が Manager の実装になる理由。
4. Employee 型の変数から Manager 固有操作を直接呼べない理由。
5. ボーナス対象であることを、従業員の種類とは別軸で扱う理由。
6. PayrollService が具体的な従業員種別をなるべく知らずに月給合計を計算できる理由。
7. ボーナス対象者だけを処理するために、どのような判定と型の扱いが必要になるか。
8. PayrollService が、受け取った配列や従業員オブジェクトに影響されない理由。
```

# 12. ヒント

必要になった場合だけ開いてください。

<details>
<summary>共通部分と差分の分け方</summary>

すべての従業員に共通する情報は、共通の親に持たせると整理しやすくなります。

一方で、月給計算や役割名のように従業員種別ごとに異なる処理は、各子クラスに任せると自然です。

</details>

<details>
<summary>ボーナス対象者の扱い</summary>

ボーナス対象であることは、正社員・管理職・アルバイトといった分類とは別の性質として考えられます。

同じ従業員でも、ボーナス対象の種類と対象外の種類があります。

そのため、「ボーナス額を計算できる」という性質を別に表すと扱いやすくなります。

</details>

<details>
<summary>PayrollService で各従業員をコピーする方法</summary>

`Employee` は共通の型ですが、実際のインスタンスは正社員・アルバイト・管理職などの具象クラスです。

`PayrollService` がすべての具象クラスを細かく判定してコピーしようとすると、種類が増えるたびに修正が必要になります。

各従業員自身に「自分と同じ内容の別インスタンスを返す」操作を持たせると、`PayrollService` 側の実装を単純にできます。

</details>

<details>
<summary>ボーナス対象者だけを処理する方法</summary>

`Employee[]` の各要素がボーナス対象者であるとは限りません。

そのため、ボーナス額を計算する前に、その従業員がボーナス対象者として扱えるかを確認する必要があります。

確認できた場合だけ、ボーナス対象者として扱えば、ボーナス額を計算できます。

</details>
